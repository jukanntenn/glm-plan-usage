  <action>
1. **Change signature of `GlmApiClient::from_env`** in client.rs from:
   ```rust
   pub fn from_env() -> Result<Self>
   ```
   to:
   ```rust
   pub fn from_env(config: &Config) -> Result<Self>
   ```
   The function still reads token and base_url from environment, but gets timeout from `config.api.timeout_ms`.

2. **Replace hardcoded timeout** on line 47: instead of `Duration::from_secs(5)`, use:
   ```rust
   .timeout(Duration::from_millis(config.api.timeout_ms))
   ```

3. **Replace hardcoded retry loop** in `fetch_usage_stats` on line 63: change from:
   ```rust
   for attempt in 0..=2 {
   ```
   to:
   ```rust
   for attempt in 0..config.api.retry_attempts {
   ```
   This gives exactly the configured number of retries: if retry_attempts = 2, you get 0 and 1 → two attempts. If retry_attempts = 0 → loop doesn't run, only one attempt is made. This matches what users expect (avoids common pitfall of adding extra attempts).

4. **Update the call site** in `glm_usage.rs` line 67: from `GlmApiClient::from_env()` to `GlmApiClient::from_env(config)` since config is already available in `get_usage_stats(&self, config: &Config)`.

5. **Do NOT change defaults** in `ApiConfig::default()` — keep 5000ms and 2 retries exactly as they are (D-05). This preserves existing behavior for users who don't configure these options.

6. **Handle zero correctly**: If `timeout_ms = 0`, ureq accepts zero as "no timeout". This is correct as passed directly.
</action>
